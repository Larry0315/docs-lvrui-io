三元运算

三元运算又称之为三目运算，是对简单条件语句的缩写，没有看错，简单的语句也要简写！

集合

集合的特性

set集合,是一个无序且不重复的元素的集合
利用集合不重复的特点,可以轻松实现列表元素的去重

创建集合

创建集合的操作与创建字典的方式很像,字典是键值对,每个元素由两个值组成,集合由一个元素组成

操作集合

  • add 向集合中添加元素
  • clear 清空集合中的所有元素
  • copy 浅拷贝
  • difference 左连接
  • difference_update 左连接更新
  • discard 移除指定元素
  • intersection 取交集
  • intersection_update 取交集并更新到调用集合中
  • isdisjoint 检查两个集合是否存在交集
  • issubset 是否是子序列 & issuperset 是否是父序列
  • pop 随机移除元素
  • remove 删除指定元素
  • symmetric_difference 对称差集
  • symmetric_difference_update 取差集并更新到调用集合中
  • union 并集
  • update 更新集合

集合的应用(去重)

在去重领域,集合是非常好用而且非常高效的方法,如果面试时有人问你,如果高效的去除一个巨型列表中重复的元素,这个时候不要装X时候自己写算法了,老老实实的说用set你就是满分啦~

深浅拷贝

深浅拷贝的问题应该放在列表和字典之后最好,无所谓啦,先塞在这里啦~~

产生深浅拷贝的问题要追溯到之前笔记文档中所说的Python数据类型的分类

  • 在Python中,数据类型分为两类:

    1. 基本数据类型(string, int, float, boolean)

    2. 引用数据类型(list, tuple, dict, set, class)

  • 基本数据类型在传递的时候是传值,也就是值得拷贝

  • 引用数据类型在传递的时候是传址,也就是内存地址的链接

知道了以上两点知识,就能很好的处理Python中深浅拷贝的问题.

函数

函数体现了面向对象编程语言封装的特性,面向对象语言的终极目标就是消除重复代码,且使用了封装 继承 多态 三种特性实现了这一伟大的目标.现在所介绍的函数,恰恰是Python封装特性的一个典型体现(有人争论Python有没有多态,我觉得讨论这点完全没有意义,因为Python本身是动态语言,Python的本身就是多态,不需要像Java/C#那样有多态的特定语法,Python在正常使用中,就是多态的!)

返回值

  • 在函数执行完毕后,会返回一个值,这就是函数的返回值.如果没有手动去指定函数的返回值,那么Python将自动返回一个None

  • 返回值可以返回任何数据类型(想想前面提到的基本数据类型和引用数据类型),但是只能返回一次且只能是一个元素

  • 返回的一个元素中可以包含多个元素,比如返回一个列表

  • 指定参数(指定为某一参数赋值,可以乱序)

  • 默认参数(不赋值时可以使用默认值)

  • 动态参数(接收任意多的参数)

函数在接收到参数后,会自动将其组成一个元组

单行函数

在Python的项目中,可能会遇到lambda的字眼,lambda就是单行函数的关键字

单行函数用来解决简单的函数运算,例如一个函数中只有一行代码就返回了的情况

注意: 在Python PEP8 规则中有明文规定(do not assign a lambda expression, use def) 意思是不要使用lambda表达式,推荐 使用def. Python语言一向以高阅读性著称,使用lambda违背了简单易懂的高阅读性原则

bytes 字符串转成字节类型

一个汉字使用utf-8编码的话占用三个字节,使用gbk编码的话占用两个字节

打开文件乱码实况再现:

如果一个文件是以utf-8的编码方式保存,每三个字节是一个汉字,再同样以utf-8的方式读取,会正确的按照3个字节一个汉字去读取文件,而如果以gbk的方式读取文件,会按照两个字节一个汉字的方式读取,所以会产生乱码的情况

用香来表示精确数字的小故事(二进制转十进制)

str 字节转成字符串

open 打开文件

  • 使用open打开文件时,可以指定encoding参数,这个参数的作用是:让Python帮助我们把字符串和二进制以指定的编码转互相转换

  • open('db', 'r', encoding='GBK')为例,Python从磁盘中读取二进制并自动以GBK编码的方式转换成字符串展示给调用者

  • 之所以会有乱码的出现,就是因为写入的编码与读取的编码不符导致的,例如使用utf-8编码来保存文件,却以gbk编码来读取就会造成乱码

以r的方式读取文件,处理流程为: Python去磁盘读取二进制代码,在内部转换成字符串展示给我们
以w x a的方式写入文件,处理流程为: 我们将字符串交给Python,Python在内部转换成二进制代码保存到磁盘中

如果以b的方式读写取文件,处理流程为:b相当于向Python声明,不需要再为我做任何转换的操作,我直接与磁盘的二进制代码进行读写,
而二进制字符会以字节的方式进行展示,因为我们已经声明不需要Python介入帮忙转换，所以我们也需要手动去把字符串转换成字节在去保存

文件的操作

Python中使用open内建函数来打开文件,Python3中,open函数支持一下打开模式:

  • “+” 表示可以同时读写某个文件

    • r+， 读写【可读，可写】
    • w+，写读【可读，可写】
    • x+ ，写读【可读，可写】
    • a+， 写读【可读，可写】

  • “b”表示以字节的方式操作

    • rb 或 r+b
    • wb 或 w+b
    • xb 或 w+b
    • ab 或 a+b

打开文件

r 只读方式打开

w 只写方式打开

x 文件存在则报错,不存在则进入只写模式

a 以追加的方式打开文件进入写入模式

r+ 最常用的读写模式

为什么r+是最常用的读写模式?

- w+虽然也是读写模式,但是w模式会先清空文件中的所有内容,一般情况下我们不需要清空文件

b 以二进制方式打开文件

如果使用带b的方式打开文件,是按照字节来读写的

如果使用不带b的方式打开文件,是按照字符来读写的

例如: 一个汉字使用utf-8编码的话,占用三个字节,使用带b方式要读取三个字节才是正确的汉字,而使用不带b的方式,读取一个字符就是正确的汉字

使用二进制的方式操作文件省去了Python为我们转换的操作

文件的其他操作

read() 读取文件，不加参数即读取全部内容，加数字类型的参数有两种情况

  • 以不带b的方式进行读操作，则读的是字符

  • 以带b的方式进行读操作，则读的是字节

seek() 操作文件指针（字节）

还有一种情况是使用a+的方式打开文件后，操作seek()指定写入位置是无效的，都会在文件末尾追加

tell() 报告当前文件指针的位置（字节）

write() 向文件中写入数据

flush() 用在关闭文件之前，强制将写入的操作写到磁盘中

truncate() 截断文件，指针之后的内容全部清空

readable() & writable() 是否可读 是否可写

readlines() 把文件所有的内容读到内存中（read()也是一次把所有内容读到内存中）

readline() 读取一行
